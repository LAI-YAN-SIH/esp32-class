二、Arduino 程式開發流程與程式架構
1 Arduino 程式開發流程
1. 編寫程式碼：首先利用Arduino IDE 編寫原始程式碼並存檔，程式的副檔名為.ino。
2. 進行編譯（Verify）及建立燒錄檔：將原始程式碼轉成機器可以看得懂的機器碼及可燒錄的.hex 檔。編譯時若發現問題的話，IDE 會顯示程式錯誤的原因，像是指令錯字、變數未宣告、資料型態錯誤等，只要依據錯誤訊息進行修改，再重新進行編譯及建立燒錄檔動作，直到出現Done compiling，即完成編譯的動作。
3. 上傳（Upload）：將燒錄檔透過USB 串列埠上傳程式至Arduino 微控制器。
4. 動作執行：當完成上述動作後，並不能保證動作正確，因為程式邏輯上的錯誤並不容易發現，只能透過動作的執行或Serial Monitor 的訊息監控來發現動作的錯誤，再依邏輯錯誤進行修改。
